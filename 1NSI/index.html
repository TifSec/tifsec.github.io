2. Introduction aux Variables

Définition d'une variable :

Une variable est un espace de stockage dans votre programme qui peut contenir des données. C'est comme une boîte pour conserver des informations.

Types de Données :

  • Chaînes de caractères (str) : Texte (ex. : nom = "Mario").
  • Entiers (int) : Nombres entiers (ex. : score = 100).
  • Flottants (float) : Nombres à virgule (ex. : temps = 3.5).
  • Booléens (bool) : Valeurs de vérité (ex. : est_vivant = True).

Exemple de Déclaration de Variables :

                nom = "Mario"
                score = 100
                temps = 3.5
                est_vivant = True

Déroulement de la Séance

1. Introduction aux Fonctions

Qu'est-ce qu'une fonction ?

Une fonction est un bloc de code réutilisable qui effectue une tâche spécifique. Les fonctions permettent de diviser le code en sections plus petites et plus faciles à gérer, d'éviter la répétition de code et de rendre les programmes plus clairs et modulaires.

Pourquoi utiliser des fonctions ?

  • Réutilisabilité : Une fois une fonction définie, elle peut être utilisée autant de fois que nécessaire dans le programme.
  • Clarté : Les fonctions permettent de structurer le code en plusieurs parties logiques.
  • Modularité : Elles facilitent la maintenance du code.

Définir une fonction :

La syntaxe pour définir une fonction est la suivante :


                def nom_de_la_fonction(paramètres):
                    # Bloc d'instructions
                    return valeur_de_retour  # (optionnel)

Exemple de Fonction Simple :


                def saluer():
                    print("Bonjour tout le monde !")

Ici, la fonction saluer() ne prend aucun paramètre et affiche simplement "Bonjour tout le monde !" lorsqu'elle est appelée.

                # Initialisation de la liste pour stocker les cas de chaque jour

1. Introduction aux Paramètres et au Retour de Valeurs

Qu'est-ce qu'un paramètre ?
Un paramètre est une variable qui est utilisée dans la définition d'une fonction pour recevoir des données lors de l'appel de la fonction.

Pourquoi passer des paramètres ?
Passer des paramètres permet de rendre les fonctions plus flexibles, car elles peuvent traiter des données différentes à chaque appel.

Retourner des valeurs :
Les fonctions peuvent également renvoyer des résultats à l'endroit où elles sont appelées à l'aide de l'instruction return.

Exemple de Fonction avec Paramètres et Retour de Valeur :


    def multiplier(a, b):
        return a * b  # Retourne le produit de a et b
    
    # Appel de la fonction
    resultat = multiplier(3, 4)
    print(resultat)  # Affiche 12

Ici, a et b sont des paramètres et la fonction retourne leur produit.

Séance 12 : Pratique - Calcul d’un Budget Familial

Objectifs de la Séance

  • Appliquer les concepts de boucles, conditions, et gestion de variables pour simuler le calcul d’un budget familial.
  • Écrire un programme permettant à l'utilisateur d'ajouter des revenus et des dépenses et de calculer le solde final.

Déroulement de la Séance

1. Introduction au Calcul du Budget Familial

Concept du budget familial : Un budget familial consiste à calculer la différence entre les revenus (salaire, allocations, etc.) et les dépenses (loyer, nourriture, loisirs). Le solde final doit indiquer si la famille est en excédent ou en déficit.

Objectif : Le programme doit permettre à l'utilisateur de saisir les revenus et les dépenses et, à la fin, afficher le solde final. Si le solde est positif, cela signifie que la famille est en excédent ; sinon, elle est en déficit.

2. Activité Pratique : Calcul d’un Budget Familial

Objectif de l'activité : Créer un programme interactif qui demande à l'utilisateur de saisir des revenus et des dépenses jusqu'à ce qu'il choisisse d'arrêter, puis affiche le solde final.

Instructions :


    total_revenus = 0
    total_depenses = 0
    
    # Saisie des revenus
    print("Entrez vos revenus :")
    while True:
        revenu = float(input("Entrez un revenu (ou 0 pour terminer) : "))
        if revenu == 0:
            break
        total_revenus += revenu  # Ajoute le revenu au total
    
    # Saisie des dépenses
    print("Entrez vos dépenses :")
    while True:
        depense = float(input("Entrez une dépense (ou 0 pour terminer) : "))
        if depense == 0:
            break
        total_depenses += depense  # Ajoute la dépense au total
    
    # Calcul du solde final
    solde = total_revenus - total_depenses
    
    # Affichage du solde
    if solde > 0:
        print(f"Votre solde final est de {solde:.2f}€. Vous êtes en excédent.")
    else:
        print(f"Votre solde final est de {solde:.2f}€. Vous êtes en déficit.")

{solde:.2f} :
solde est une variable qui contient le solde final, calculé en soustrayant les dépenses des revenus.
.2f est une spécification de format qui indique comment le nombre doit être affiché :
.2 : Affiche le nombre avec deux chiffres après la virgule.
f : Spécifie que le nombre doit être affiché comme un nombre à virgule flottante.
Par exemple, si solde vaut 123.4567, l'expression {solde:.2f} afficherait 123.46 (le nombre est arrondi à deux chiffres après la virgule)

Explications du Code :

  • Boucle pour les revenus : L'utilisateur entre ses revenus dans une boucle while qui continue jusqu'à ce qu'il entre 0 pour terminer. Chaque revenu est ajouté à la variable total_revenus.
  • Boucle pour les dépenses : Le processus est identique pour les dépenses, avec une autre boucle while qui continue jusqu'à ce que l'utilisateur entre 0.
  • Calcul du solde final : Le solde est calculé en soustrayant les dépenses totales des revenus totaux. Le programme affiche ensuite si le solde est positif (excédent) ou négatif (déficit).

2. Testez votre programme :

3. Exercices Supplémentaires

Exercice 1 : Catégorisation des Dépenses

Objectif de l'exercice :
Dans cet exercice, nous allons modifier le programme original de gestion du budget familial pour permettre à l'utilisateur d'entrer ses dépenses par catégorie (par exemple, loyer, nourriture, loisirs). Ensuite, le programme affichera un récapitulatif des dépenses par catégorie, permettant à l'utilisateur de voir combien il a dépensé dans chaque domaine.

Ce que l'on va faire :

  1. Demander à l'utilisateur la catégorie de chaque dépense.
  2. Stocker les dépenses dans un dictionnaire où chaque clé représente une catégorie (ex : "loyer", "nourriture") et chaque valeur est la somme des dépenses pour cette catégorie.
  3. Afficher un récapitulatif des dépenses par catégorie à la fin.

Étapes détaillées

  1. Initialiser un Dictionnaire pour les Dépenses
    Un dictionnaire est une structure de données Python qui permet de stocker des paires clé-valeur. Dans cet exercice, les clés seront les catégories de dépenses (comme "loyer" ou "nourriture"), et les valeurs seront les montants dépensés dans ces catégories.
    depenses = {}
  2. Demander à l'utilisateur d'entrer une catégorie et un montant pour chaque dépense
    Nous allons utiliser une boucle pour demander à l'utilisateur d'entrer des catégories de dépenses et les montants associés.
  3. Mettre à jour le dictionnaire à chaque nouvelle dépense
    À chaque fois que l'utilisateur saisit une nouvelle dépense, le programme doit vérifier si la catégorie existe dans le dictionnaire. Si c'est le cas, le programme ajoute simplement le montant à la valeur existante. Sinon, il crée une nouvelle entrée pour cette catégorie.
  4. Afficher un récapitulatif des dépenses par catégorie
    Une fois que l'utilisateur a terminé d'entrer ses dépenses (par exemple, en entrant 0 comme montant), le programme affiche un récapitulatif qui liste toutes les catégories et les montants dépensés dans chacune.

Code Complet : Catégorisation des Dépenses


    # Initialisation du dictionnaire des dépenses
    depenses = {}
    
    # Demande à l'utilisateur d'entrer ses dépenses par catégorie
    print("Entrez vos dépenses par catégorie (ex: loyer, nourriture, loisirs).")
    print("Entrez 0 comme montant pour terminer.")
    
    while True:
        categorie = input("Entrez la catégorie de dépense (ou 'fin' pour terminer) : ")
        
        if categorie.lower() == 'fin':  # Permet de sortir de la boucle avec 'fin'
            break
        
        montant = float(input(f"Entrez le montant pour la catégorie '{categorie}' : "))
        
        if montant == 0:  # Si le montant est 0, on arrête l'entrée des dépenses
            break
        
        # Si la catégorie existe déjà dans le dictionnaire, on ajoute au montant
        if categorie in depenses:
            depenses[categorie] += montant
        else:
            depenses[categorie] = montant  # Sinon, on crée la catégorie avec ce montant
    
    # Récapitulatif des dépenses par catégorie
    print("\nRécapitulatif des dépenses par catégorie :")
    for categorie, montant in depenses.items():
        print(f"{categorie.capitalize()} : {montant:.2f} €")

Explication du Code

  1. Initialisation du dictionnaire depenses :
    On commence par créer un dictionnaire vide pour stocker les catégories et les montants associés.
  2. Boucle while pour entrer les catégories et montants :
    La boucle continue tant que l'utilisateur ne choisit pas de quitter (en entrant 'fin').
  3. Mise à jour du dictionnaire :
    Si la catégorie existe déjà, on met à jour la valeur en ajoutant le montant actuel au total existant. Sinon, on crée une nouvelle entrée.
  4. Affichage du récapitulatif :
    Après que l'utilisateur ait terminé d'entrer les dépenses, le programme affiche un récapitulatif des montants dépensés pour chaque catégorie.

Exercice 2 : Prévisions

Objectif de l'Exercice : Prévisions pour le Mois Suivant

Dans cet exercice, vous allez ajouter une fonctionnalité qui permet à l'utilisateur d'entrer une prévision pour le mois suivant, indiquant s'il prévoit une augmentation ou une diminution de ses revenus ou de ses dépenses. Ensuite, vous calculerez un solde estimé pour le mois suivant en fonction de ces prévisions.

Étapes pour Ajouter la Fonctionnalité de Prévision

  1. Saisie des Revenus et Dépenses Actuels
  2. Saisie des Prévisions pour le Mois Suivant
  3. Calcul du Solde Estimé pour le Mois Suivant
  4. Affichage du Solde Estimé

Exemple de Code pour Gérer les Prévisions


    # Étape 1 : Saisie des revenus et dépenses actuels
    total_revenus = 0
    total_depenses = 0
    
    print("Entrez vos revenus pour le mois actuel :")
    while True:
        revenu = float(input("Entrez un revenu (ou 0 pour terminer) : "))
        if revenu == 0:
            break
        total_revenus += revenu  # Ajoute le revenu au total
    
    print("Entrez vos dépenses pour le mois actuel :")
    while True:
        depense = float(input("Entrez une dépense (ou 0 pour terminer) : "))
        if depense == 0:
            break
        total_depenses += depense  # Ajoute la dépense au total
    
    # Calcul du solde actuel
    solde = total_revenus - total_depenses
    
    # Affichage du solde actuel
    if solde > 0:
        print(f"Votre solde actuel est de {solde:.2f}€. Vous êtes en excédent.")
    else:
        print(f"Votre solde actuel est de {solde:.2f}€. Vous êtes en déficit.")
    
    # Étape 2 : Saisie des prévisions pour le mois suivant
    print("\nPrévisions pour le mois suivant :")
    
    # Prévision des revenus (en pourcentage)
    pourcentage_revenus = float(input("Entrez le pourcentage de variation des revenus (ex: +10 ou -5) : "))
    revenus_prevision = total_revenus + (total_revenus * pourcentage_revenus / 100)
    
    # Prévision des dépenses (en pourcentage)
    pourcentage_depenses = float(input("Entrez le pourcentage de variation des dépenses (ex: +10 ou -5) : "))
    depenses_prevision = total_depenses + (total_depenses * pourcentage_depenses / 100)
    
    # Étape 3 : Calcul du solde estimé pour le mois suivant
    solde_estime = revenus_prevision - depenses_prevision
    
    # Étape 4 : Affichage du solde estimé pour le mois suivant
    if solde_estime > 0:
        print(f"Votre solde estimé pour le mois suivant est de {solde_estime:.2f}€. Vous serez en excédent.")
    else:
        print(f"Votre solde estimé pour le mois suivant est de {solde_estime:.2f}€. Vous serez en déficit.")
