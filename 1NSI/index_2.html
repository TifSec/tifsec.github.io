Séance 1 : Représentation des entiers naturels en binaire

Objectifs :

  • Comprendre la base 2 (binaire) et les positions.
  • Passer de base 10 → base 2 et base 2 → base 10 (méthodes variées).
  • Connaître la base 16 (hexadécimale) et ses liens avec la base 2.
  • Estimer le nombre de bits nécessaires pour coder un entier.
  • Éviter les erreurs classiques (zéros non significatifs, regroupements hex, etc.).

Partie théorique

1) Pourquoi la base 2 ?

Un ordinateur manipule des circuits électroniques qui ont deux états stables (tension haute/basse). On code donc l’information avec des bits :
bit ∈ {0, 1}
octet = 8 bits → permet de représenter 256 valeurs (0 à 255).
