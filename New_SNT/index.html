🔒 Mission 1 : S'introduire dans le réseau

🔎 Ce que vous allez voir

Dans cette mission, vous allez explorer le fonctionnement d’un site web du point de vue d’un navigateur et analyser comment il communique avec un serveur.

  • Le modèle client-serveur : Comprendre la relation entre un client (navigateur) et un serveur (site web).
  • Les requêtes HTTP : Observer comment les données sont échangées entre le navigateur et le serveur.
  • Les différents types de fichiers : Identifier les fichiers chargés pour construire une page web (HTML, CSS, JavaScript, images, etc.).
  • Les réponses du serveur : Comprendre les statuts des requêtes HTTP (200, 404, 500, etc.).

📌 Pourquoi c'est important ?

  • Comprendre comment un site web répond aux requêtes : Chaque interaction repose sur des requêtes HTTP envoyées au serveur.
  • Identifier des points de faiblesse : Certains fichiers ou requêtes peuvent exposer des informations sensibles exploitables.
  • Développer des compétences essentielles en cybersécurité.

🛠️ Outils

  • Navigateur Web : Chrome ou Firefox.
  • Console de développement : Accessible avec F12, onglet "Réseau".
  • Site cible : example.com

🔄 Mission

1️⃣ Ouvrez le site cible et activez la console réseau

Appuyez sur F12 pour ouvrir la console de développement et allez dans l’onglet "Réseau" pour observer le trafic réseau en direct.

Pourquoi ? : Cela permet de visualiser les requêtes envoyées au serveur et leurs réponses.

2️⃣ Analysez les requêtes envoyées par le client au serveur

Rechargez la page (F5) et observez les requêtes générées.

Repérez les colonnes importantes :

  • Nom : indique l'URL de la ressource demandée.
  • Méthode : GET (récupération de données) ou POST (envoi de données)
  • Statut : 200 (succès), 404 (page non trouvée), 500 (erreur serveur).
  • Type : HTML, CSS, JavaScript, image, JSON, etc.

Pourquoi ? : En identifiant le type de requêtes et les erreurs éventuelles, vous pouvez repérer des vulnérabilités ou des fichiers sensibles mal protégés.

3️⃣ Identifiez le type de contenu reçu en retour

Triez les requêtes par type (HTML, CSS, images, scripts...).

Cliquez sur une requête pour voir sa réponse détaillée (ex. code source HTML retourné).

Pourquoi ? : Cela permet de comprendre la structure du site et d'identifier des fichiers qui pourraient contenir des informations exploitables (ex. un fichier JavaScript révélant des API internes).

4️⃣ Trouvez des informations exploitables

Observez les en-têtes HTTP :

  • Certains révèlent des informations sur le serveur utilisé (ex. Apache, Nginx, PHP).
  • Recherchez des identifiants de session ou des cookies non sécurisés.

Notez les fichiers JavaScript accessibles :

Certains contiennent des endpoints d’API ou des clés d’authentification laissées visibles par erreur.

Pourquoi ? : Ces informations peuvent être utilisées pour comprendre la structure du site et identifier d’éventuelles failles de sécurité.

🌟 Résultats attendus

  • Liste des requêtes HTTP avec leur statut (ex. 200, 404).
  • Identification des types de fichiers récupérés depuis le serveur.
  • Analyse des URL des fichiers potentiellement exploitables.

💡 Conseil final : Notez vos observations et préparez-vous pour la mission suivante. Bonne chance, agents de ShadowHack !
